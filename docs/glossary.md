# Glossary

This glossary provides definitions for key terms and concepts used throughout the book. Terms are listed alphabetically.

## A

### AI (Artificial Intelligence)
The simulation of human intelligence processes by machines, especially computer systems, including learning, reasoning, and self-correction.

### API (Application Programming Interface)
A set of protocols, tools, and definitions for building application software and enabling communication between different software components.

## B

### Build
The process of converting source code and assets into a deployable format. In Docusaurus, this creates static HTML, CSS, and JavaScript files.

## C

### CI/CD (Continuous Integration/Continuous Deployment)
A method to frequently deliver apps to customers by introducing automation into the stages of app development.

### Component
A reusable, self-contained piece of code that encapsulates specific functionality or UI elements. In React/Docusaurus, components are typically written as functions or classes.

### Constitution
The foundational document that defines core principles, standards, and governance for the project.

## D

### Deployment
The process of making an application available for use, typically by uploading it to a web server or hosting platform.

### Docusaurus
An open-source static site generator optimized for creating documentation websites, built with React.

## E

### Example
A code sample or demonstration that illustrates a concept or technique discussed in the content.

## F

### Frontmatter
Metadata placed at the beginning of a Markdown file, typically in YAML format, that provides information about the document.

## G

### GitHub Pages
A free static site hosting service provided by GitHub that serves websites directly from a GitHub repository.

### Glossary
A collection of terms and their definitions, serving as a single source of truth for terminology.

## H

### Hero Section
The prominent section at the top of a webpage, typically containing a headline, description, and call-to-action.

## I

### Interactive Example
A code example that can be run and modified directly in the browser, often using tools like CodeSandbox or StackBlitz.

## L

### Learning Objective
A measurable statement describing what a reader should be able to do after completing a chapter or section.

### Lighthouse
An open-source tool from Google for measuring and improving web page quality, including performance, accessibility, and SEO.

## M

### Markdown
A lightweight markup language for creating formatted text using a plain-text editor.

### MDX
An extension of Markdown that allows you to use JSX (React components) directly in your Markdown files.

### Metadata
Data that provides information about other data. In web development, this includes page titles, descriptions, and keywords for SEO.

## N

### Navigation
The system of menus, links, and controls that allow users to move through a website or application.

## O

### Open Graph
A protocol that enables any web page to become a rich object in a social graph, used for social media sharing.

## P

### PHR (Prompt History Record)
A document that captures the context, inputs, and outputs of AI-assisted work for traceability and learning.

### Prerequisite
Knowledge, skills, or completed chapters required before starting a new chapter or section.

## R

### React
A JavaScript library for building user interfaces, particularly single-page applications.

### Responsive Design
An approach to web design that makes web pages render well on a variety of devices and window sizes.

## S

### SEO (Search Engine Optimization)
The practice of increasing the quantity and quality of traffic to your website through organic search engine results.

### Sidebar
A vertical navigation menu typically displayed on the left or right side of a web page.

### Spec (Specification)
A detailed document that defines the requirements, scope, and objectives for a feature or chapter.

### Static Site
A website composed of fixed content that is delivered to users exactly as stored, without server-side processing.

## T

### Template
A predefined structure or format that serves as a starting point for creating similar documents or files.

### TypeScript
A typed superset of JavaScript that compiles to plain JavaScript, adding optional static typing.

## U

### UI (User Interface)
The space where interactions between humans and machines occur, including all visual and interactive elements.

### UX (User Experience)
The overall experience a person has when interacting with a product, especially in terms of how easy and satisfying it is.

## V

### Version Control
A system that records changes to files over time so that you can recall specific versions later. Git is the most common version control system.

## W

### WCAG (Web Content Accessibility Guidelines)
International guidelines for making web content more accessible to people with disabilities.

### WebP
A modern image format that provides superior compression for images on the web.
