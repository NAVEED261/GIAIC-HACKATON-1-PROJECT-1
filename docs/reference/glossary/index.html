Action

Also known as: ROS Action, Action Server

A ROS 2 communication pattern for long-running tasks that provides feedback during execution and allows cancellation. Actions combine request-response (like services) with ongoing feedback updates.

Related chapters:
  • Module 1: Actions
  • Multi-Node Systems
Learn more:
  • ROS 2 Actions Documentation

Actuator

Also known as: Motor, Servo

A mechanical device that converts electrical signals into physical motion. Common types include DC motors, servo motors, and stepper motors used to control robot joints and movements.

Related chapters:
  • Module 2: Robot Modeling
Learn more:
  • Actuator Types Overview

Adaptive Monte Carlo Localization

Also known as: AMCL, MCL

A probabilistic localization algorithm that uses particle filters to estimate a robot's position within a known map. It adapts the number of particles based on localization confidence.

Related chapters:
  • Module 2: Navigation
Learn more:
  • AMCL ROS Package

AGI

Also known as: Artificial General Intelligence

A hypothetical form of artificial intelligence that possesses the ability to understand, learn, and apply knowledge across a wide range of tasks at a human-like level.

Related chapters:
  • Module 4: VLA Models

Bag File

Also known as: ROSbag, ROS Bag

A file format used in ROS to record and playback topic data. Useful for debugging, testing, and creating reproducible scenarios without re-running live systems.

Related chapters:
  • Module 1: ROS 2 Fundamentals
Learn more:
  • ROS 2 Bag Documentation

Base Link

Also known as: base_link, Robot Base

The primary coordinate frame attached to the robot's base, serving as the reference point for all other frames in the robot's transformation tree (TF tree).

Related chapters:
  • Module 1: ROS 2 Fundamentals
  • Module 2: URDF Modeling

BERT

Also known as: Bidirectional Encoder Representations from Transformers

A transformer-based language model developed by Google that uses bidirectional training to understand context from both left and right of a word in sentences.

Related chapters:
  • Module 4: VLA Models
Learn more:
  • BERT Paper

Camera Intrinsics

Also known as: Calibration Matrix, K Matrix

Parameters describing a camera's internal characteristics including focal length, optical center, and lens distortion. Essential for 3D reconstruction and computer vision tasks.

Related chapters:
  • Module 3: Perception Pipeline

Capstone Project

Also known as: Final Project

The culminating project integrating all four course modules (ROS 2, Digital Twin, Isaac Sim, VLA) into a complete autonomous humanoid system capable of natural language control.

Related chapters:
  • Week 13: Capstone Integration

Cartesian Space

Also known as: Task Space, Operational Space

The 3D workspace defined by X, Y, Z coordinates and orientation (roll, pitch, yaw) where the robot end-effector operates, as opposed to joint space.

Related chapters:
  • Module 2: Manipulation

CLIP

Also known as: Contrastive Language-Image Pretraining

An OpenAI model that learns visual concepts from natural language supervision, enabling zero-shot image classification and vision-language understanding.

Related chapters:
  • Module 4: VLA Models
Learn more:
  • CLIP Paper

Colcon

Also known as: Colcon Build System

The build system used in ROS 2 for compiling packages. It supports multiple build types (CMake, Python setuptools, ament) and parallel builds for faster compilation.

Related chapters:
  • Module 1: ROS 2 Fundamentals
Learn more:
  • Colcon Tutorial

Collision Detection

Also known as: Collision Checking

The process of determining whether objects in a simulated or real environment intersect or come into contact. Critical for motion planning and safety.

Related chapters:
  • Module 2: Gazebo Simulation
  • Module 3: Isaac Sim

Costmap

Also known as: Occupancy Grid, Navigation Costmap

A 2D grid representation of the environment where each cell has a cost value indicating traversability. Used by navigation algorithms to plan collision-free paths.

Related chapters:
  • Module 2: Navigation

CUDA

Also known as: Compute Unified Device Architecture

NVIDIA's parallel computing platform and API that allows developers to use GPUs for general-purpose processing, essential for deep learning and physics simulation.

Related chapters:
  • Setup: Hardware Requirements
  • Module 3: Isaac Sim
Learn more:
  • CUDA Toolkit

DDS

Also known as: Data Distribution Service, RTPS

The middleware protocol used by ROS 2 for inter-process communication. Provides real-time publish-subscribe messaging with quality-of-service guarantees.

Related chapters:
  • Module 1: ROS 2 Architecture
Learn more:
  • DDS Specification

Degrees of Freedom

Also known as: DOF

The number of independent parameters that define a robot's configuration. A 6-DOF manipulator can position and orient its end-effector freely in 3D space.

Related chapters:
  • Module 2: Kinematics

Digital Twin

Also known as: Virtual Twin, Simulation Model

A virtual replica of a physical system that mimics its behavior, appearance, and dynamics. Used for testing, training, and validation before deploying to real hardware.

Related chapters:
  • Module 2: Digital Twin Simulation

Direct Kinematics

Also known as: Forward Kinematics, FK

The process of calculating the end-effector position and orientation given joint angles. The opposite of inverse kinematics.

Related chapters:
  • Module 2: Kinematics

Domain Randomization

Also known as: Sim-to-Real Transfer

A technique for training models in simulation with randomized parameters (lighting, textures, physics) to improve transferability to real-world environments.

Related chapters:
  • Module 2: Unity ML-Agents

Embodied AI

Also known as: Physical AI, Embodied Intelligence

AI systems that interact with the physical world through sensors and actuators, as opposed to purely virtual AI. Includes robots, autonomous vehicles, and smart devices.

Related chapters:
  • Introduction

End Effector

Also known as: Gripper, Tool

The device at the end of a robotic arm that interacts with the environment, such as a gripper, suction cup, or welding torch.

Related chapters:
  • Module 2: Manipulation

Extended Kalman Filter

Also known as: EKF

A nonlinear version of the Kalman filter used for state estimation by linearizing around the current estimate. Commonly used for sensor fusion and robot localization.

Related chapters:
  • Module 3: Sensor Fusion

Fuse.js

Also known as: Fuzzy Search

A JavaScript library for lightweight fuzzy-search functionality, used in this textbook for the glossary search component with approximate string matching.

Related chapters:
  • Setup: Glossary Search
Learn more:
  • Fuse.js Documentation

Gazebo

Also known as: Gazebo Simulator, Ignition Gazebo

An open-source 3D robot simulator that provides physics simulation, sensor models, and integration with ROS. Used for testing algorithms before hardware deployment.

Related chapters:
  • Module 2: Digital Twin Simulation
Learn more:
  • Gazebo Website

Grasp Planning

Also known as: Grasping, Grasp Synthesis

The process of determining optimal gripper poses and trajectories to securely grasp objects. Involves force analysis, contact modeling, and stability evaluation.

Related chapters:
  • Module 3: Manipulation

Humanoid Robot

Also known as: Anthropomorphic Robot, Android

A robot with a body shape resembling the human form, typically including a torso, head, two arms, and two legs. Examples include Atlas, Optimus, and ASIMO.

Related chapters:
  • Module 4: Humanoid Integration

IMU

Also known as: Inertial Measurement Unit

A sensor that measures linear acceleration and angular velocity using accelerometers and gyroscopes. Essential for robot orientation and motion estimation.

Related chapters:
  • Module 1: Sensors
  • Module 3: Sensor Fusion

Inverse Kinematics

Also known as: IK

The process of calculating joint angles required to achieve a desired end-effector position and orientation. Often has multiple solutions or no solution.

Related chapters:
  • Module 2: Kinematics
  • Module 3: Manipulation

Isaac Sim

Also known as: NVIDIA Isaac Sim, Omniverse Isaac

NVIDIA's GPU-accelerated robotics simulator built on Omniverse, providing photorealistic rendering, accurate physics, and synthetic data generation for perception training.

Related chapters:
  • Module 3: NVIDIA Isaac Sim
Learn more:
  • Isaac Sim Documentation

Jetson

Also known as: NVIDIA Jetson, Jetson Orin

NVIDIA's line of embedded computing boards designed for edge AI applications. The Jetson Orin Nano is recommended for this course's edge deployment option.

Related chapters:
  • Setup: Jetson Orin Nano
Learn more:
  • Jetson Products

Joint Space

Also known as: Configuration Space

The space defined by robot joint angles (q1, q2, ... qn), as opposed to Cartesian space. Motion planning often occurs in joint space.

Related chapters:
  • Module 2: Kinematics

Kalman Filter

Also known as: KF, Linear Quadratic Estimation

An optimal state estimation algorithm that combines predictions from a model with noisy sensor measurements to estimate system state with minimal error variance.

Related chapters:
  • Module 3: Sensor Fusion

Lidar

Also known as: LiDAR, Laser Scanner

A sensor that measures distance by illuminating targets with laser light and measuring reflections. Produces 2D or 3D point clouds for mapping and obstacle detection.

Related chapters:
  • Module 3: Perception Pipeline

LLM

Also known as: Large Language Model

A deep learning model trained on vast text corpora to understand and generate human language. Examples include GPT-4, Claude, and PaLM used in VLA architectures.

Related chapters:
  • Module 4: VLA Models

Localization

Also known as: Robot Localization

The process of determining a robot's position and orientation within a known map or environment using sensor data and probabilistic algorithms.

Related chapters:
  • Module 2: Navigation
  • Module 3: Sensor Fusion

Message

Also known as: ROS Message, Topic Message

A data structure in ROS that defines the format of information exchanged between nodes via topics. Defined in .msg files with primitive and composite types.

Related chapters:
  • Module 1: Nodes and Topics

Motion Planning

Also known as: Path Planning, Trajectory Planning

The process of finding a collision-free path for a robot to move from a start to goal configuration. Algorithms include RRT, PRM, and A*.

Related chapters:
  • Module 2: Navigation
  • Module 3: Manipulation

MoveIt

Also known as: MoveIt2, Motion Planning Framework

An open-source motion planning framework for ROS that provides inverse kinematics, collision checking, and manipulation primitives for robot arms.

Related chapters:
  • Module 2: Manipulation
Learn more:
  • MoveIt 2 Documentation

Navigation2

Also known as: Nav2, ROS 2 Navigation

The official ROS 2 navigation stack providing mobile robot navigation capabilities including localization, path planning, and obstacle avoidance.

Related chapters:
  • Module 2: Navigation
Learn more:
  • Navigation2 Docs

Node

Also known as: ROS Node, Computational Node

An executable process in ROS that performs computation. Nodes communicate via topics, services, and actions to form distributed robotic systems.

Related chapters:
  • Module 1: ROS 2 Fundamentals

Occupancy Grid

Also known as: Grid Map

A 2D probabilistic map representation where each cell indicates the probability that the space is occupied by an obstacle. Used for navigation and SLAM.

Related chapters:
  • Module 2: Mapping

Odometry

Also known as: Odom, Wheel Odometry

The estimation of a robot's change in position over time using sensor data such as wheel encoders, IMU, or visual features. Accumulates error over distance.

Related chapters:
  • Module 1: Sensors
  • Module 2: Localization

Omniverse

Also known as: NVIDIA Omniverse

NVIDIA's platform for 3D simulation and collaboration built on Universal Scene Description (USD). Isaac Sim is built on top of Omniverse.

Related chapters:
  • Module 3: Isaac Sim Setup
Learn more:
  • Omniverse Platform

Particle Filter

Also known as: Sequential Monte Carlo, SMC

A probabilistic localization algorithm that represents the robot's pose distribution with a set of weighted particles. Used in AMCL for localization.

Related chapters:
  • Module 2: Localization

PhysX

Also known as: NVIDIA PhysX

NVIDIA's real-time physics engine used in Isaac Sim for accurate rigid body dynamics, collision detection, and contact simulation with GPU acceleration.

Related chapters:
  • Module 3: Isaac Sim

Point Cloud

Also known as: 3D Point Cloud, PCL

A collection of 3D points representing object surfaces or environments, typically captured by lidar or depth cameras. Used for object detection and mapping.

Related chapters:
  • Module 3: Perception Pipeline

Pose

Also known as: 6D Pose, Position and Orientation

The combination of a 3D position (x, y, z) and 3D orientation (roll, pitch, yaw or quaternion) that fully describes an object's location in space.

Related chapters:
  • Module 1: TF2
  • Module 3: Pose Estimation

Publish-Subscribe

Also known as: Pub-Sub, Topic Communication

A messaging pattern where publishers send data to topics without knowing subscribers, and subscribers receive data without knowing publishers. Core ROS 2 pattern.

Related chapters:
  • Module 1: Nodes and Topics

QoS

Also known as: Quality of Service

Policies in ROS 2 DDS that control message reliability, durability, and history. Different QoS profiles optimize for real-time vs reliable communication.

Related chapters:
  • Module 1: Advanced Topics
Learn more:
  • ROS 2 QoS Docs

Quaternion

Also known as: Unit Quaternion

A mathematical representation of 3D orientation using four numbers (w, x, y, z) that avoids gimbal lock. Preferred over Euler angles in robotics.

Related chapters:
  • Module 1: TF2
  • Module 2: Kinematics

Ray Tracing

Also known as: Path Tracing, RTX

A rendering technique that simulates light propagation for photorealistic images. Isaac Sim uses GPU ray tracing for sensor simulation and synthetic data.

Related chapters:
  • Module 3: Isaac Sim

RGB-D Camera

Also known as: Depth Camera, RGBD Sensor

A camera that captures both color (RGB) and depth information for each pixel, enabling 3D reconstruction. Examples include Intel RealSense and Azure Kinect.

Related chapters:
  • Module 3: Perception

Robot Operating System

Also known as: ROS, ROS 2

An open-source framework for robot software development providing communication infrastructure, hardware abstraction, and package management. ROS 2 is the current version.

Related chapters:
  • Module 1: ROS 2 Fundamentals
Learn more:
  • ROS 2 Official Site

RRT

Also known as: Rapidly-Exploring Random Tree

A sampling-based motion planning algorithm that builds a tree of random configurations to find paths in high-dimensional spaces. Fast but produces jagged paths.

Related chapters:
  • Module 2: Path Planning

RViz

Also known as: RViz2, ROS Visualization

A 3D visualization tool for ROS that displays sensor data, robot models, transforms, and planned paths. Essential for debugging and monitoring robots.

Related chapters:
  • Module 1: ROS 2 Tools

Sensor Fusion

Also known as: Multi-Sensor Integration

The process of combining data from multiple sensors (IMU, GPS, lidar, camera) to produce more accurate and reliable state estimates than any single sensor.

Related chapters:
  • Module 3: Sensor Fusion

Service

Also known as: ROS Service, Request-Response

A ROS 2 communication pattern for synchronous request-response interactions. A client sends a request and blocks until receiving a response from the server.

Related chapters:
  • Module 1: Services

SLAM

Also known as: Simultaneous Localization and Mapping

The problem of building a map of an unknown environment while simultaneously localizing the robot within that map. Fundamental problem in mobile robotics.

Related chapters:
  • Module 2: Mapping and Localization

Synthetic Data

Also known as: Simulated Data

Artificially generated training data from simulation rather than real-world collection. Isaac Sim excels at generating labeled synthetic images for perception training.

Related chapters:
  • Module 3: Isaac Sim
  • Module 4: VLA Training

TF2

Also known as: Transform Library, tf2_ros

ROS 2's coordinate frame transformation system that manages the relationships between different reference frames in a robot system over time.

Related chapters:
  • Module 1: ROS 2 Fundamentals
Learn more:
  • TF2 Tutorials

Topic

Also known as: ROS Topic

A named bus over which nodes exchange messages in a publish-subscribe pattern. Topics enable asynchronous, many-to-many communication in ROS 2.

Related chapters:
  • Module 1: Nodes and Topics

Transformer

Also known as: Attention Mechanism

A deep learning architecture using self-attention mechanisms, foundational to modern language models (GPT, BERT) and increasingly used in vision and robotics.

Related chapters:
  • Module 4: VLA Architecture
Learn more:
  • Attention Is All You Need

Unity ML-Agents

Also known as: ML-Agents, Unity Machine Learning

An open-source toolkit for training intelligent agents in Unity environments using reinforcement learning, imitation learning, and neuroevolution.

Related chapters:
  • Module 2: Digital Twin Simulation
Learn more:
  • ML-Agents GitHub

URDF

Also known as: Unified Robot Description Format

An XML format for describing robot models including links, joints, visual meshes, collision geometry, and inertial properties. Standard for ROS robots.

Related chapters:
  • Module 2: Robot Modeling
Learn more:
  • URDF Tutorials

Vision-Language-Action

Also known as: VLA, VLA Model, Embodied AI Model

A class of models that integrate vision, language understanding, and action prediction to enable robots to follow natural language commands for manipulation tasks.

Related chapters:
  • Module 4: VLA Models

Visual Servoing

Also known as: Vision-Based Control

A control technique that uses visual feedback from cameras to guide robot motion, enabling tasks like object tracking and precision manipulation.

Related chapters:
  • Module 3: Vision-Based Control

VRAM

Also known as: Video RAM, GPU Memory

Memory dedicated to the graphics card for storing textures, framebuffers, and computation data. Isaac Sim requires 12GB+ VRAM for high-fidelity simulation.

Related chapters:
  • Setup: Hardware Requirements

Workspace

Also known as: ROS Workspace, Colcon Workspace

A directory structure containing ROS 2 packages that are built together. Consists of src/ (source code), build/ (build artifacts), install/ (binaries), and log/ folders.

Related chapters:
  • Module 1: ROS 2 Setup

YOLO

Also known as: You Only Look Once, YOLOv8

A family of real-time object detection models that process images in a single forward pass. YOLOv8 is commonly used for robotic perception tasks.

Related chapters:
  • Module 3: Object Detection
Learn more:
  • YOLOv8 Documentation

Zero-Shot Learning

Also known as: ZSL

The ability of a model to recognize or perform tasks it wasn't explicitly trained on by leveraging semantic knowledge. VLA models exhibit zero-shot manipulation capabilities.

Related chapters:
  • Module 4: VLA Models
